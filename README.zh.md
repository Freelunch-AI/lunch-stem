## �🔍 概述

这是一个不断发展的 *STEM*（科学、技术、工程和数学）知识库，旨在通过社区的努力进行审查和改进。它可以被人类和AI代理使用和改进。

它的理想用例是**在你对某个 *STEM* 主题有初步理解后，深入研究该主题（和相关主题）**（你可以通过谷歌搜索或AI助手轻松获得初步理解）。

对于这个用例，它应该比默认的谷歌搜索/AI深度研究更有组织性和更高质量（信噪比）。

目标是，稍后通过创建一个 *lunchSTEM MCP 服务器*，使AI代理能够轻松地将其用作工具。

## 🎯 这是为谁准备的？

- **学生** 寻找补充学习材料
- **专业人士** 想要深化他们的STEM知识
- **研究人员** 需要有组织的参考材料
- **教育工作者** 寻找教学资源
- **自学者** 进行独立学习

## 🗑️ 内容删除和作者归属请求

我们付出了巨大努力来检测和删除受版权保护的（不可分发的）内容，并识别剩余材料的作者/出版商/大学。由于文件数量庞大，无法对每个文件进行手动审查（但我们欢迎社区通过开启issues和PRs来帮助我们）。
1. 我们运行脚本删除包含除以下扩展名之外的任何其他扩展名的文件：`.pdf`、`.txt`、`.md`、`.ipynb`、`.json`
2. 我们运行脚本自动检测文档中与版权相关的关键词并删除此类文档
3. 我们运行脚本自动删除学术论文。
4. 我们手动将每个书籍pdf替换为其链接。
5. 我们运行脚本为每个剩余的pdf自动创建作者归属文件（`.source.json`），包含信息如：作者、源链接、是否修改等。字段的默认值是 `null`，除了 `changes_were_made` 字段的默认值是 `False`。当在pdf本身中找不到信息时使用默认值。

然而，我们无法保证这个过程的完美性，因此，如果你发现任何受版权保护的内容或没有适当作者归属数据的内容，请开启issue和/或提交PR和/或发送邮件至 bruno.c.scaglione@gmail.com。我们的目标是在24小时内解决问题。请参考 `CONTRIBUTING.md` 文件了解相关指南。

> **内容删除请求的简化协议（推荐使用而非 _DMCA_）**
   > 1. 阅读 CONTRIBUTING.md 查看issue指南
   > 2. 开启一个 *内容删除请求* issue
   > 3. 发送邮件至 bruno.c.scaglione@gmail.com，主题为 "[lunchSTEM] 内容删除请求: #GITHUB_ISSUE_NUMBER_PLACEHOLDER"，说明：你是谁、需要删除的内容路径以及你开启的具体issue链接。

<br>

> 这个选项比 *DMCA* 通知__更快更友好__。如果我们收到多个 *DMCA* 通知，项目有被从 *Github* 删除的风险（即使在删除内容后），许多可能从中受益的人将受到影响。

**数字千年版权法 (*DMCA*) 合规：** 我们遵守数字千年版权法 (DMCA)。对于正式的删除请求，请遵循 *DMCA* 流程。

## 如何使用

1.  **打开一个终端**：要打开终端，请使用您操作系统的搜索功能（在Mac上按Windows键或Cmd + Space），然后输入“terminal”、“command prompt”或“Windows Terminal”并选择该应用程序。

2.  **使用git克隆仓库**（此命令将在您当前目录中创建一个`lunch-stem`文件夹）。

      ```bash
      git clone https://github.com/Freelunch-AI/lunch-stem.git
      ```
      > [!NOTE]
      > 如果您使用的是Windows，重要的是要在顶级目录内进行克隆，以避免与创建过长的文件路径相关的潜在错误。Windows通常的文件路径最大长度为260个字符。

3. 设置项目

   **对于Linux**

   启用脚本执行

   ```bash
   chmod +x scripts/setup
      ```

   ```bash
   source scripts/setup
      ```

   **对于Windows：**

   在终端会话中启用脚本执行

   ```powershell
   scripts/setup.ps1
      ```

3.  进入`lunch-stem`文件夹。

    ```bash
    cd lunch-stem
    ```

4.  **浏览`ai2f`文件夹**

5.  **下载pdf文件：**

   - 对于`.pdf.dvc`文件

      **通过以下方式下载特定的pdf文件：**

      ```bash
      lunch files "<first/dvc/file/path/placeholder.pdf.dvc>" "[second/dvc/file/path/placeholder.pdf.dvc]"
      ```

      您可以放入多个文件路径，只有第一个是必需的。

      此命令将获取`.pdf`文件并将其放在您当前目录中.

      如果您想将文件放在与`pdf.dvc ` 文件相同的位置，则使用：

      - _注意3：_ 其他文件类型（例如`.txt`）应直接打开，无需使用lunch CLI。
      - _注意4：_ 如果存在`.web.txt`，则不应尝试此命令，只需将`.web.txt`内的链接复制并粘贴到浏览器中即可。我们稍后将实现一个`lunch get`来从网络上获取文件。

      ```bash
      lunch folder "<folder/path/placeholder>"
      ```

   - 对于`pdf.web.txt`文件：

      简单地打开文件并跟随其中的网络链接。

## 📁 目录结构和命名约定

- **`__Loopback`** 目录包含路径太长的文件。在这些文件的位置创建了一个指针 `.sym.txt` 文件，指向位于 `__Loopback` 目录内的实际文件。这些指针txt文件遵循此命名约定：`文件名.文件扩展名.sym.txt`，位于实际文件应该在的同一目录中。

- 根目录的 **`to_add.txt`** 文件包含稍后要包含在lunchSTEM中的材料链接。

- 以 **MEGA** 开头的文件或文件夹表示聚合材料（聚合关于特定主题的大量链接的材料）。

- 以 **Awesome** 开头的文件或文件夹表示超高质量内容。
